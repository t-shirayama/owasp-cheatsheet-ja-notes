## 概要

秘密の質問は推測、調査、漏えい、再利用に弱く、強い認証要素として扱うべきではありません。使う場合は限定的なリスク低減策が必要です。

## 要点

- 秘密の質問を主要な認証要素にしない。
- 推測しにくく変化しにくい質問を選ぶ。
- 回答をパスワード同様に保護する。

## 実装時の注意点

- この要約はASVS Indexでの利用を前提にした実装者向け整理です。詳細確認時は原文を参照してください。
- 関連する認証、認可、ログ、入力検証、暗号、通信保護の管理策と組み合わせて適用します。
