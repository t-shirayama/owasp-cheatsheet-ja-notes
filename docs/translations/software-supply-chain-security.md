## 日本語訳

ソフトウェアは真空中で開発されるものではなく、IDE、内部コード、サードパーティライブラリ、バージョン管理、ビルドツール、CI/CD、構成管理、パッケージマネージャーなどからなるソフトウェアサプライチェーン (SSC) に組み込まれています。依存関係の脆弱性、VCS の誤設定、CI/CD の侵害、成果物の改ざんなど、1つの構成要素の欠陥がサプライチェーン全体を危険にします。

## 脅威の概要

サプライチェーン脅威は広範です。dependency confusion、上流プロバイダの侵害、コード署名証明書の窃取、CI/CD システム悪用などが含まれます。脅威は大きく、ソースコード脅威、ビルド環境脅威、依存関係脅威、デプロイ・ランタイム脅威に分けられます。大規模な供給者が侵害されると、下流の利用者にも影響が連鎖します。

## 一般的な対策

開発、ビルド、バージョン管理、配布環境では、最小権限、職務分離、MFA、認証情報ローテーション、平文保存禁止、ソース管理へのシークレットコミット禁止を徹底します。

VCS、ビルドツール、配布機構、成果物リポジトリ、実行環境は、認証試行、構成変更、異常挙動を検知できるイベントをログ記録し監視します。ログは SIEM やログ集約基盤に集め、検知と対応に使える状態にします。

SAST、DAST、SCA、コンテナイメージスキャナなどのセキュリティ自動化は、手動レビューの代替ではありませんが、規模と一貫性を提供します。これらのツール自体も保守、保護、正しく設定する必要があります。

## ソースコード脅威の緩和

コードレビューは、意図しない脆弱性と悪意あるコードの両方を見つけるための低コストな検知・抑止策です。レビューは、対象技術とセキュアコーディングに経験のあるピアが、マージ前に実施し、結果を記録します。

VCS は強いアクセス制御、ログ、監視に加え、保護ブランチ、マージポリシー、レビュー必須、署名付きコミットなどの機能を活用します。VCS にシークレットをコミットしてはいけません。IDE、開発プラグイン、拡張機能も攻撃経路になり得るため、信頼できるものだけを使い、組織の資産インベントリに含めます。

## 依存関係脅威の緩和

サードパーティサービス、製品、OSS コンポーネントを取り込む前に、供給者と具体的な提供物を評価します。保守状況、成熟度、利用実績、リリース品質、メンテナ数、依存関係更新、テストカバレッジ、セキュリティ関連テスト、ドキュメント、脆弱性報告プロセス、対応速度、ライセンス整合性を確認します。

依存関係は導入時だけでなく SDLC 全体で監視します。SBOM を生成・消費し、CI/CD へ組み込むことが推奨されます。NVD、OSV、CISA KEV などの情報源や OWASP Dependency Check、retire.js などのツールを使い、既知脆弱性を自動的に監視します。SAST は内部コードだけでなく OSS コンポーネントの分析にも使えますが、誤検知と見逃しがあるため手動検証を伴わせます。lockfile や version pinning により、検証済みの特定バージョンへ依存関係を固定します。

## ビルド脅威の緩和

ビルドツール、バージョン、プラグインのインベントリを自動収集し、脆弱性情報やベンダーアドバイザリを監視します。ビルド環境は分離されたネットワークに配置し、未使用サービスを無効化し、DLP などで持ち出しを検知・防止します。CI/CD パイプライン設定やビルドスクリプトはバージョン管理し、レビューとマージルールを適用します。

成果物にはコード署名を適用し、利用側は署名を検証します。署名基盤自体が侵害されると下流への攻撃に使われるため、署名鍵と署名プロセスを強く保護します。プライベート成果物リポジトリを使い、承認済み成果物だけを取り込めるようにし、迂回を防ぎます。

provenance は、成果物がどこで、いつ、どのように生成されたかを示す検証可能なメタデータです。ローカル開発環境ではなくビルドプラットフォームで生成し、攻撃者が偽造しにくく、builder と成果物を結び付けられる十分な情報を含めます。ビルドは再利用環境ではなく、隔離された一時環境で実行し、完了後に破棄します。ユーザーが制御できるビルドパラメータは最小化または排除します。

## デプロイとランタイム脅威

ビルド完了後の成果物を安全と仮定してはいけません。最終バイナリやコンテナイメージに対して、シークレット露出、未承認コンポーネント、完全性、脆弱性を検査します。デプロイ後も、コード依存関係、コンテナイメージ、Web サーバー、OS コンポーネント、構成変更を含めて継続監視します。正確で最新のコンポーネントインベントリが前提になります。
