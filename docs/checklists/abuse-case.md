## 開発チェックリスト

- [ ] 主要な業務フローごとに悪用ケースを定義する。
- [ ] 悪用ケースから必要な認可、入力検証、監査ログを導出する。
- [ ] 悪用ケースを脅威モデリングとテストケースへ反映する。
- [ ] 高リスク操作の悪用パターンをレビューする。
- [ ] リリース前に悪用ケースベースの検証を実施する。
