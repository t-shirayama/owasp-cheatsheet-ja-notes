## 章別チェック観点

- 記録すべきイベント、ログ形式、保護、監視、保持、アラートを確認する。
- エラー応答が過度な内部情報を露出せず、調査に必要な情報を安全に残すか確認する。
- ログ対象、ログ属性、除外データ、ログ保護、障害時の振る舞いを設計段階で決める。
- ログが攻撃対象になる前提で、機密性、完全性、可用性、説明責任を保護する。
